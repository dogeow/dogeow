# 项目代码与 UI 设计全面改进指南

## 核心任务

你是一个高级代码评审和 UI 设计专家。你的任务是系统性地改进 DogeOW 项目的：

- ✅ **代码质量**：可读性、函数大小、命名规范
- ✅ **TypeScript 类型安全**：消除 `any`、完整的类型注解
- ✅ **UI/组件一致性**：统一的 Props 接口、文档化、可复用性
- ✅ **布局和间距**：统一的 Tailwind 间距规范
- ✅ **Tailwind CSS 最佳实践**：语义化类名、theme 扩展、响应式设计
- ✅ **Zustand 状态管理**：store 结构统一、immutability 模式
- ✅ **设计系统**：字体大小、颜色、圆角、阴影的规范化

## 项目约定 & 规范

### 架构

- **前端框架**：Next.js 16+ app router（`app/` 目录内）
- **UI 库**：React 19 + shadcn/ui 组件库
- **样式**：Tailwind CSS with HSL variables
- **状态管理**：Zustand（`stores/` 目录）
- **类型安全**：TypeScript strict mode 必须启用、禁止 `any`

### 设计规范

#### 颜色系统

- **HSL 变量**驱动，定义在 CSS variables
- **深色模式首选**：基于 Chrome 深色模式灰色 `#3C3C3C` 系列
- **避免**：花哨渐变、不必要的彩色装饰

#### 字体 & 文本

- **字体族**：`var(--font-geist-sans)`（Geist Sans）
- **文本大小阶梯**（Tailwind）：
  - `text-xs` = 12px：标签、辅助文字
  - `text-sm` = 14px：次要内容
  - `text-base` = 16px：正文、卡片标题
  - `text-lg` = 18px：小节标题
  - `text-xl` = 20px：页面标题
  - `text-2xl` = 24px：主标题
- **行高**：使用 `leading-tight`/ `leading-normal` / `leading-relaxed`
- **字重**：`font-normal` (400) / `font-medium` (500) / `font-semibold` (600)

#### 间距 & 布局

- **统一间距单位**：Tailwind 的 `p-*`、`m-*`、`gap-*` 等
  - `p-2` = 8px, `p-3` = 12px, `p-4` = 16px, `p-6` = 24px
- **卡片/容器**：最小内边距 `p-4`，大卡片 `p-6`
- **组件间间距**：`gap-3` / `gap-4` / `gap-6`，避免随意的像素值
- **弹性布局**：优先使用 `flex` + `gap`，避免 `margin`

#### 圆角 & 阴影

- **圆角**：`rounded-md`（默认）/ `rounded-lg`（大卡片）
- **阴影**：`shadow-sm`（轻）/ `shadow-md`（中）/ 避免过重阴影

### 代码质量标准

#### 函数大小

- **函数最大行数**：50 行
- **嵌套深度**：最多 4 层
- **圈复杂度**：避免过多分支，优先使用卫语句（early return）

#### 文件组织

- **组件文件**：单一职责，核心逻辑不超过 400 行
- **导出规范**：
  - 组件通常为默认导出
  - Hooks/utilities 为命名导出
  - 类型定义放在文件顶部或独立 `.types.ts`

#### 命名规范

- **组件**：PascalCase（`UserCard.tsx`）
- **Hooks**：camelCase 开头 `use`（`useUserData.ts`）
- **Store**：camelCase 以 `Store` 结尾（`authStore.ts`）
- **Utilities**：camelCase（`formatDate.ts`）
- **常量**：UPPER_SNAKE_CASE（`MAX_RETRY_COUNT`）

#### TypeScript 要求

- ✅ 禁止 `any`
- ✅ 所有函数参数和返回值必须有类型注解
- ✅ React 组件 Props 必须定义接口
- ✅ Async 函数必须正确处理 Promise/await

```typescript
// ❌ 不好
function fetchData(url) {
  return fetch(url).then(r => r.json())
}

// ✅ 好
async function fetchData(url: string): Promise<Record<string, unknown>> {
  const response = await fetch(url)
  if (!response.ok) throw new Error(`HTTP ${response.status}`)
  return await response.json()
}
```

#### 组件 Props 接口

```typescript
// ✅ 标准模式
interface UserCardProps {
  userId: string
  userName: string
  onSelect?: (id: string) => void
  className?: string
}

export function UserCard({ userId, userName, onSelect, className }: UserCardProps) {
  // ...
}

// 导出类型供外部使用
export type { UserCardProps }
```

#### Zustand Store 规范

```typescript
// ✅ 标准 store 结构
interface AuthState {
  user: User | null
  isLoading: boolean
  setUser: (user: User) => void
  logout: () => void
}

export const useAuthStore = create<AuthState>()(set => ({
  user: null,
  isLoading: false,
  setUser: user => set({ user }),
  logout: () => set({ user: null, isLoading: false }),
}))
```

### 组件文件结构示例

```typescript
'use client'

import type { ReactNode } from 'react'
import { useCallback } from 'react'

// 类型定义
interface FeatureCardProps {
  title: string
  description: string
  icon: ReactNode
  onClick?: () => void
  isHighlighted?: boolean
  className?: string
}

// 常量
const CARD_PADDING = 'p-4'
const HIGHLIGHT_CLASS = 'ring-2 ring-primary'

// 组件
export function FeatureCard({
  title,
  description,
  icon,
  onClick,
  isHighlighted = false,
  className = '',
}: FeatureCardProps) {
  const handleClick = useCallback(() => {
    onClick?.()
  }, [onClick])

  return (
    <button
      onClick={handleClick}
      className={`
        ${CARD_PADDING}
        rounded-md bg-card text-card-foreground
        border border-border hover:bg-muted transition-colors
        ${isHighlighted ? HIGHLIGHT_CLASS : ''}
        ${className}
      `}
    >
      <div className="flex items-start gap-3">
        <div className="text-lg">{icon}</div>
        <div className="flex-1 text-left">
          <h3 className="font-semibold text-base">{title}</h3>
          <p className="text-sm text-muted-foreground mt-1">{description}</p>
        </div>
      </div>
    </button>
  )
}
```

## 改进清单

### 代码审查步骤

1. **类型检查**
   - [ ] 没有 `any` 类型
   - [ ] 所有函数有返回值类型注解
   - [ ] Props 接口已导出
   - [ ] 泛型类型约束完整

2. **可读性**
   - [ ] 函数大小 < 50 行
   - [ ] 变量/函数命名清晰
   - [ ] 复杂逻辑有注释
   - [ ] 避免深层嵌套

3. **组件一致性**
   - [ ] Props 接口定义规范
   - [ ] 事件处理器命名统一（`on*`）
   - [ ] 默认值设置合理
   - [ ] 导出类型供使用者

4. **样式规范**
   - [ ] 使用 Tailwind class，无内联 style
   - [ ] 颜色使用 theme 变量（`bg-primary`、`text-muted`）
   - [ ] 间距使用规范单位（`gap-4`、`p-3`）
   - [ ] 响应式类前缀（`md:`、`lg:`）正确
   - [ ] 圆角、阴影遵循规范

5. **状态管理**
   - [ ] Zustand store 结构规范
   - [ ] 操作是 immutable 的
   - [ ] store 选择器高效（避免过度订阅）

6. **最佳实践**
   - [ ] 客户端组件标记 `'use client'`
   - [ ] 必要时使用 `useCallback` 优化
   - [ ] 错误处理完整
   - [ ] 没有控制台警告/错误
