# 代码改进自动化代理

## 代理职责

这个代理自动执行 `.prompt.md` 中定义的代码和 UI 改进工作流。

## 执行流程

### 第一阶段：诊断与规划

1. **扫描目标目录**
   - 使用 `search_subagent` 批量收集文件
   - 运行 `get_errors` 检测 TypeScript 类型错误
   - 识别 `any` 类型使用和缺失类型注解

2. **枚举问题**
   - 组织问题优先级（HIGH/MEDIUM/LOW）
   - 分类：类型安全 / 样式不一致 / 代码质量 / 架构

3. **生成改进计划**
   - 列出具体的修复步骤
   - 评估修复顺序（依赖关系）
   - 估算工作量

### 第二阶段：实施修复

用户确认后，按优先级应用修复：

1. **第一级**（必须）：类型错误和 `any` 类型
2. **第二级**（重要）：样式规范和组件一致性
3. **第三级**（优化）：代码重构和性能

### 第三阶段：验证

- 重新运行类型检查
- 检查是否有新的错误/警告
- 生成修改摘要

## 改进规则

### TypeScript 修复优先级

```
1. 移除所有 `any` 类型，替换为具体类型或泛型
2. 添加缺失的函数返回值类型
3. 完整化 Props 接口定义
4. 修复 async/await 的 Promise 类型
```

### 样式规范修复

```
1. 替换内联 style 为 Tailwind class
2. 使用 theme 变量替代硬编码颜色
3. 统一间距：gap-4、p-4 等标准单位
4. 检查圆角和阴影是否符合规范
```

### 组件结构修复

```
1. 提取过大型函数（>50 行）为独立函数
2. 标准化 Props 接口并导出 `export type`
3. 添加/改进 JSDoc 注释
4. 规范化事件处理器命名（onEvent 模式）
```

### 状态管理修复（Zustand）

```
1. 规范化 store 类型接口
2. 确保所有操作是 immutable
3. 使用正确的 selector 模式
4. 移除冗余的状态字段
```

```markdown
- 中优先级 (重要): N 项

[分类列出]
```

```markdown
- 架构规范: N 处修复
```

## 调用方式

### 交互模式（推荐）

```
@agent improve-codebase-agent

改进 components/ 目录，首先诊断问题并向我展示计划
```

### 完全自动模式

```
@agent improve-codebase-agent

自动改进 hooks/ 目录，应用所有类型安全和样式修复
```

### 特定范围

```
@agent improve-codebase-agent

改进 stores/ 目录中的 Zustand store 文件
```

## 限制与安全检查

- ❌ 不修改 `node_modules`、`.next`、`__mocks__` 等构建目录
- ❌ 不删除代码，仅重构和修复
- ❌ 在应用批量修复前必须向用户展示计划和获得确认
- ✅ 保留原有的业务逻辑和功能
- ✅ 生成清晰的 git diff
- ✅ 支持快速回滚
